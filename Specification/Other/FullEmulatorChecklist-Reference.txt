Full emulator checklist (NESDEV and ines spec)
============================================
Цель: "открыл ROM — всё работает". Чеклист без смены архитектуры.

---- 1. CARTRIDGE / LOADER ----
  [OK] iNES signature, PRG/CHR from header
  [OK] Header cleanup (DiskDude, demiforce, Ni03) — ines header cleanup
  [OK] iNES 2.0: mapper byte 8, PRG/CHR byte 9, TV region byte 12
  [OK] Trainer 512 bytes when byte 6 bit 2
  [OK] Mirroring: byte 6 bit 0/1, bit 3 four-screen, mapper 7 single-screen
  [OK] PRG/CHR clamp by file size
  [--] UNIF / FDS / NSF — не реализовано

---- 2. REGION / TIMING ----
  [OK] Region from header (Flags 9, 10, iNES 2.0 byte 12)
  [OK] PreferRegion override (WPF: имя файла (E)/(Europe)/(PAL))
  [OK] TvRegionProfile: NTSC 262 scanlines, PAL 312, frame interval, CPU clock, APU frame counter
  [OK] PPU SetRegion(lastScanline), APU SetRegion(profile), Clock.Interval
  [--] Dendy (312 lines, NMI at 291) — опционально, отдельный регион

---- 3. CONSOLE LOOP ----
  [OK] Update(): while (frame not done) { Cpu.Step(); PpuStep(cycles); ApuStep(cycles); }
  [OK] PPU: 3 steps per CPU cycle (NES 3 PPU cycles per CPU cycle)
  [OK] Frame done by Ppu.TotalFrames change
  [OK] ColdBoot/WarmBoot on insert

---- 4. CPU (6502) ----
  [OK] Instruction set, addressing, cycle count per opcode
  [OK] NMI/IRQ (PPU VBlank, mapper IRQ, APU frame/DMC)
  [OK] Address space: RAM, PPU, APU, input, cartridge
  [OK] 0x4014 OAM DMA write → PPU FillOAM
  [OK] OAM DMA cycle penalty: после записи в $4014 добавляем 511 циклов PPU/APU (513 всего − 2 на инструкцию), NESDEV
  [--] DMC DMA cycle steal (APU тянет байты из PRG) — у нас DMC не до конца (комментарий в коде)

---- 5. PPU ----
  [OK] 341 dots × (262|312) scanlines, VBlank at 241
  [OK] BG + sprites, 8x8/8x16, mirroring (H/V/four/single)
  [OK] OAM, OAM DMA (FillOAM), sprite 0 hit, sprite overflow (Status)
  [OK] Scroll/nametable (ScrollFormula), $2000-$2007
  [OK] Scanline callback for mapper (MMC3 IRQ)
  [--] Pre-render scanline (-1), точный цикл NMI (1-3 циклов задержки) — для пиксель-перфекта
  [--] Эмуляция открытой шины PPU при отключённом рендере — опционально

---- 6. APU ----
  [OK] Pulse ×2, Triangle, Noise; регион (NTSC/PAL) через TvRegionProfile
  [OK] Frame counter 4-step (quarter/half frame), length counter, envelope, sweep
  [OK] 48 kHz output, SamplesPerFrame по региону
  [--] DMC (DPCM) — частично (канал есть, вывод закомментирован, нет DMA steal)

---- 7. INPUT ----
  [OK] $4016/$4017, strobe, 8 buttons (A,B,Select,Start,U,D,L,R)
  [--] Zapper, Famicom expansion — по желанию

---- 8. MAPPERS ----
  Реализовано в ASD.NES: 0 (NROM), 1 (MMC1), 2 (UxROM), 3 (CNROM), 4 (MMC3), 7 (AxROM),
  11 (Color Dreams), 34 (BNROM/Nina), 66 (GxROM/MHROM), 71 (Camerica), 79 (AVE).
  Fallback: неподдерживаемый маппер → NROM (ROM грузится, игра может глючить).

  Приоритет для добавления (частые ROM): 5 (MMC5), 9 (MMC2), 10 (MMC4), 13, 16, 18, 19, 21-26 (VRC), 28, 32, 33, 36, 37, 38, 48, 64, 65, 69, 78, 80, 82, 85, 88, 94, 206.

---- 9. ЧТО МОЖНО ДОБАВИТЬ БЕЗ ЛОМКИ АРХИТЕКТУРЫ ----
  • Новые Board (мапперы) по одному — по тому же паттерну (Board, SetPRG/SetCHR, Read/Write, scanline при необходимости).
  • DMC: включить вывод, реализовать DMA steal (1–4 цикла на байт при fetch).
  • Dendy: третий TvRegion с lastScanline=311, SLStartNMI=291 (и константы APU при необходимости).
  • Документировать неподдерживаемый маппер при fallback (например, свойство Cartridge.MapperNumber и в лог/UI "Mapper X not supported, running as NROM").

---- 10. НЕ ПЕРВООЧЕРЕДНО ----
  • Точный PPU по циклам (pre-render, задержка NMI) — большой объём.
  • UNIF/FDS/NSF загрузка.
  • Zapper, expansion port.
  • Сейвы/батарейка (SRAM) — в Board при необходимости.

Итог: ядро (загрузка, регион, цикл, CPU, PPU, APU, ввод, мапперы 0,1,2,3,4,7,11,34,66,71,79) выровнено с NESDEV/ines в пределах текущей архитектуры. По мере появления конкретных ROM — добавлять мапперы, при желании DMC.
