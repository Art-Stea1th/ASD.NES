ROM compatibility: почему ~2/3 игр не запускаются и план правок
================================================================
(По NESDEV, списку мапперов и типичным багам эмуляторов.)

1) Поддерживаемые мапперы в ASD.NES
-----------------------------------
Сейчас: 0 (NROM), 1 (MMC1), 2 (UxROM), 3 (CNROM), 4 (MMC3), 7 (AxROM),
        11 (Color Dreams), 34 (BNROM/NINA), 66 (GxROM), 71 (Camerica), 79 (NINA-003/006).
Неизвестный маппер → запасной вариант NROM (mapper 0). Для игр с mapper 2/5/9/10 и т.д.
это даёт неверную раскладку банков → зависание или чёрный экран.

2) Частые мапперы в коммерческих ROM
------------------------------------
0, 1, 2, 3, 4, 7 покрывают большинство лицензионных игр. Дополнительно часто:
5 (MMC5), 9 (MMC2, Punch-Out), 10 (MMC4), 19 (Namco 163), 24/26 (VRC6), 69 (Sunsoft 5B), 85 (VRC7).
Игры из папки пользователя (Megaman, Castlevania, Contra, Battletoads, etc.) в основном
используют 0, 1, 2, 3, 4, 7 — они уже в списке поддержки. Значит причины сбоев скорее:
- баги в реализации MMC1/MMC3 (или других),
- тайминг IRQ MMC3,
- зеркалирование/размеры PRG-CHR по заголовку.

3) Известные проблемы по NESDEV
-------------------------------
MMC1: Часты ошибки в переключении CHR (4K/8K), порядок битов в serial register,
      зеркалирование. Неверный CHR bank → графический мусор или краш.
MMC3: IRQ — счётчик тикает по rising edge PPU A12, а не раз за сканлайн.
      $C001: запись обнуляет счётчик и выставляет перезагрузку на СЛЕДУЮЩЕМ тике A12,
      а не сразу. $C000 не меняет текущий счётчик, только значение при следующей перезагрузке.
      Иначе игры, полагающиеся на точный IRQ по сканлайну (скролл, эффекты), глючат или не запускаются.
Заголовок: очистка (DiskDude, demiforce, Ni03) уже есть; iNES 2.0 mapper/PRG/CHR — есть.
Неизвестный маппер: тихая подстановка NROM даёт неверное поведение для всех не-NROM игр.

4) План действий (строго по спекам)
-----------------------------------
A. Сканирование: запустить scan_nes_headers.ps1 по папке с ROM (e:\...\roms),
   получить распределение мапперов. Добавить в тесты опциональную проверку:
   «для каждого маппера из скана либо есть поддержка, либо явный skip».
B. Неизвестный маппер: не подставлять NROM молча. Варианты: (1) выбросить исключение
   с номером маппера при загрузке; (2) или добавить мапперы-заглушки по мере необходимости.
   Решение — уточнить с пользователем (вилка в поведении).
C. MMC3: привести IRQ к спекам — перезагрузка по $C001 на «следующем такте» (следующий вызов
   OnScanline или точный A12), не сразу. Реализовать счётчик по A12 или консервативно:
   один тик за сканлайн, но reload только на следующем сканлайне после $C001.
D. MMC1: сверить с NESDEV формулы CHR 4K/8K и serial register; поправить при расхождении.
E. Тесты: переписать/добавить по официальным спекам (NESDEV); при необходимости вилка
   (например, опция «строгий маппер» или «тесты только по NESDEV»).

5) Что сделать в первую очередь
--------------------------------
- Реализовать пункт C (MMC3 IRQ reload по спекам).
- Реализовать пункт B (не молча подставлять NROM для неизвестного маппера — хотя бы лог/ошибка).
- Запустить сканер по папке ROM и добавить в план список мапперов, которых не хватает.
- Затем по списку добавить мапперы (5, 9, 10, …) или заглушки с понятным поведением.
