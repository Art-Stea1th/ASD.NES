================================================================================
Сравнение: Nintendulator vs ASD.NES (по NESDEV, ReferenceEmulators, коду)
================================================================================

| Критерий                    | Nintendulator                    | ASD.NES                              |
|-----------------------------|----------------------------------|--------------------------------------|
| Язык / платформа            | C++, Windows                     | C#, .NET Standard 1.1 (Core), WPF     |
| Цель                        | Максимальная точность, в ущерб скорости | Учебный/практический эмулятор, тесты |
| Расположение исходников     | e:\...\nintendulator0980_src\    | Репозиторий ASD.NES                  |


--- CPU ---
| Модель выполнения           | Цикл-точный, переписан под точность | По инструкциям: Step() → циклы, затем PPU/APU |
| Синхронизация с PPU         | По циклам                        | PPU.Step() × (cpuCycles × 3) за один шаг CPU   |
| OAM DMA                      | Учтено                           | 513 циклов (511 после записи) — NESDEV        |


--- PPU ---
| Модель                      | Цикл-по-циклу по документации    | По циклам: 341 dot/scanline, Scan + PPUCore   |
| Сканлайны NTSC              | 262 (SLEndFrame)                 | 262 (0..260), lastScanline 260                 |
| Сканлайны PAL               | 312                              | 312 (0..311)                                  |
| NMI (VBlank)                | Строка 241 (NTSC/PAL), 291 (Dendy) | Строка 241, точка 1                           |
| Денди                       | REGION_DENDY, отдельные константы | Нет отдельного региона                        |


--- APU ---
| Регион                      | SetRegion() → APU::SetRegion()   | SetRegion(region) → TvRegionProfile, frame counter |
| Счётчик кадра               | Регион-специфичный              | Шаги NTSC/PAL из TvRegionProfile (3728, 7456, …)   |
| Выход                       | По документации (в работе)     | 48 кГц, циклы→сэмплы через CpuClockHz/sampleRate   |


--- Картридж / заголовок ---
| iNES 2.0                    | byte 7 & 0x0C == 0x08; TV byte 12 | То же; mapper |= (byte8 & 0x0F)<<8; PRG/CHR byte 9 |
| Регион из заголовка         | Только iNES 2.0 (byte 12)       | Flags 9 (bit 0), 10 (bits 0–1), iNES 2.0 byte 12   |
| Очистка заголовка           | Не описана в референсе          | DiskDude, demiforce, Ni03 (как FCEUX)               |
| Мапперы                     | Внешние DLL (nintendulator-mappers) | Встроенные в Core                                |
| Список мапперов             | 0–250+ через плагины            | 0, 1, 2, 3, 4, 7, 11, 34, 66, 71, 79               |
| Неизвестный маппер          | Нет в ядре (плагин не найден)   | Запасной вариант NROM (mapper 0)                 |
| Конфликт шины (UxROM/CNROM)| Зависит от реализации в DLL    | Опции EmulationOptions (UxROMBusConflict, CNROMBusConflict) |


--- Регион / тайминг ---
| NTSC CPU clock              | —                                | 1 789 773 Hz (TvRegionProfile)                    |
| PAL CPU clock               | —                                | 1 662 607 Hz                                      |
| Частота кадра NTSC          | —                                | ~60.0988 Hz                                       |
| Частота кадра PAL           | —                                | 50 Hz                                             |
| Переопределение региона     | Не из заголовка iNES 1.0         | PreferRegion (например по имени файла (E)/PAL)    |


--- Зеркалирование ---
| Управление                  | В PPU/плагине                    | PPUAddressSpace.NametableMirroring, header byte 6, mapper 7 → SingleScreen |


--- Тесты и отладка ---
| Юнит-тесты                  | Нет в открытой поставке          | 205 тестов (ASD.NES.Tests)                        |
| Ссылка на эталон            | Сам позиционируется как эталон   | FCEUX/Nintendulator в ReferenceEmulators-EmuFolder |


--- Итог по «диффу» ---
• Nintendulator: цикл-точное ядро, мапперы вынесены в DLL, много мапперов, Денди.
• ASD.NES: встроенные мапперы (ограниченный набор), запасной вариант NROM, опции конфликта шины,
  единый TvRegionProfile, регион из заголовка и переопределение, 262/312 сканлайнов,
  PPU по циклам (341×сканлайн), тесты по NESDEV.
